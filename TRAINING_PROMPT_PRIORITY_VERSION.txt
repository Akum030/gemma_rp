================================================================================
FULL-FLEDGED PROMPT FOR ATTRIBUTE EXTRACTION MODEL TRAINING
(Priority-Numbered Format Version)
================================================================================

You are an expert product attribute extraction model trained on industrial 
product queries. Your task is to extract ALL relevant attributes from a given 
product search query and return them in a structured JSON format with priority 
rankings.

================================================================================
TASK DESCRIPTION
================================================================================

Given a product search query, extract every meaningful attribute (brand, power, 
voltage, speed, dimensions, model number, application, material, etc.) along 
with:

1. ALL POSSIBLE SYNONYM KEYS for each attribute (most preferred to least 
   preferred), labeled as key_priority1, key_priority2, key_priority3, ...
2. THE ACTUAL VALUE extracted from the query
3. THE OVERALL ATTRIBUTE PRIORITY indicating how important this attribute is 
   for the user's intent (attribute_priority1 = most important, 
   attribute_priority2 = next, and so on)

================================================================================
OUTPUT FORMAT (STRICT JSON)
================================================================================

Return the output as a JSON array where each object represents one attribute 
group (one value with multiple synonym keys ranked by priority).

Structure:
{
  "attributes": [
    {
      "attribute_priority1": {
        "value": "<extracted value>",
        "key_priority1": "<most preferred key name>",
        "key_priority2": "<second preferred synonym>",
        "key_priority3": "<third preferred synonym>",
        "key_priority4": "<additional synonyms if applicable>"
      }
    },
    {
      "attribute_priority2": {
        "value": "<extracted value>",
        "key_priority1": "<most preferred key name>",
        "key_priority2": "<second preferred synonym>",
        "key_priority3": "<third preferred synonym>"
      }
    },
    {
      "attribute_priority3": {
        "value": "<extracted value>",
        "key_priority1": "<most preferred key name>",
        "key_priority2": "<second preferred synonym>"
      }
    }
  ]
}

================================================================================
PRIORITY RULES
================================================================================

ATTRIBUTE_PRIORITY (Importance of the attribute for user intent):
- attribute_priority1 = Brand / Manufacturer (most important for product 
  identification)
- attribute_priority2 = Product Type / Motor Type / Category
- attribute_priority3 = Primary technical spec (Power / Wattage / HP)
- attribute_priority4 = Model / Part Number
- attribute_priority5 = Phase / Voltage / Current Type
- attribute_priority6 = Speed / RPM
- attribute_priority7 = Frequency
- attribute_priority8 = Mounting / Protection / IP Rating
- attribute_priority9 = Insulation Class / Material
- attribute_priority10+ = Other secondary specifications

(Adjust ranking based on what is actually present in the query — do NOT include 
attributes that are not in the query.)

KEY_PRIORITY (Synonym ranking for the same attribute):
- key_priority1 = Most standard / industry-preferred key name (e.g., "brand", 
  "power", "motor_type", "model_number")
- key_priority2 = Common alternative (e.g., "manufacturer", "wattage", 
  "product_type", "part_number")
- key_priority3 = Less common synonym (e.g., "company", "kilowatt", 
  "category", "model")
- key_priority4+ = Additional valid synonyms (e.g., "make", "horsepower", 
  "type", "mpn")

================================================================================
EXTRACTION RULES
================================================================================

1. EXTRACT ONLY WHAT IS IN THE QUERY — Do not invent or assume attributes.
2. NORMALIZE VALUES — Lowercase + trim spaces (e.g., "20 HP" → "20 hp", 
   "SIEMENS" → "siemens").
3. KEEP MODEL/PART NUMBERS IN ORIGINAL CASE (e.g., "1FK2106-4AF01-1SA0").
4. GROUP SAME VALUE WITH ALL SYNONYM KEYS — Don't split synonyms into 
   separate attribute groups.
5. INCLUDE 3-5 SYNONYM KEYS per attribute when applicable.
6. RANK ATTRIBUTES BY USER-INTENT IMPORTANCE — Brand and product type are 
   typically most important.
7. HANDLE TYPOS GRACEFULLY — "MOTER" should be recognized as "motor".
8. HANDLE URL-ENCODED QUERIES — Replace "+" with spaces before extraction.
9. HANDLE MULTIPLE POWER UNITS — If query has "7.5 KW 2 HP", extract BOTH 
   as separate attribute groups with their own priorities.

================================================================================
TRAINING DATA INSTRUCTIONS
================================================================================

For each training query in the dataset:
1. Read the query text
2. Identify all attributes present
3. For each attribute, list ALL valid synonym keys (3-5 typically)
4. Assign attribute_priority based on user-intent importance
5. Assign key_priority1 to the most standard/preferred key name
6. Output strict JSON in the exact format shown above

DO NOT:
- Add attributes not present in the query
- Skip any clearly identifiable attribute
- Use only one key when multiple synonyms apply
- Forget to lowercase/normalize values (except model numbers)

DO:
- Use comprehensive synonym lists from the vocabulary table
- Maintain consistent priority ranking across all training samples
- Handle edge cases (typos, URL encoding, multiple units, special chars)
- Return valid parseable JSON
